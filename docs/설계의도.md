# QuickNoteObsidian 설계 의도 문서

> 작성일: 2026-03-31
> 상태: 설계 확정, 구현 전

---

## 1. 프로젝트 목적

macOS 메뉴바에 상주하며, SuperWhisper 음성 인식 결과를 자동으로 Obsidian 노트로 생성하는 앱.

**핵심 워크플로우:**
```
글로벌 단축키 → SuperWhisper 녹음 → meta.json 저장 → 앱이 감지 → Obsidian 노트 생성
```

---

## 2. 해결하려는 문제

- Obsidian에 음성으로 빠르게 아이디어를 기록하고 싶지만, 글로벌 단축키 + 음성 입력 + 자동 노트 생성을 하나로 묶는 기존 솔루션이 없음
- Voice Inbox for Obsidian: 테스트 결과 불합격
- macOS 내장 받아쓰기: 영어를 한글 음차 변환 (SuperWhisper → 슈퍼 위스퍼), "조우"→兆(1,000,000,000,000) 같은 치명적 오류

---

## 3. 음성 엔진 비교 테스트 결과 (2026-03-31)

테스트 문장 3종으로 맥북 내장 받아쓰기 vs SuperWhisper Standard 비교:

| 평가 항목 | 맥북 | SuperWhisper | 승자 |
|---|---|---|---|
| 한국어 고유명사 | ✅ 케이건 드라카 | ❌ K-건 드라카 | 맥북 |
| 영어 용어 보존 | ❌ 전부 한글 음차 | ✅ 영어 그대로 유지 | SuperWhisper |
| 버전/숫자 정확도 | ❌ 15점3.2, 조→兆 | ✅ 15.3.2, 조우했다 | SuperWhisper |
| 띄어쓰기 | ❌ 불필요한 공백 다수 | ✅ 일관되게 양호 | SuperWhisper |
| 문맥 이해 | ❌ 조우→兆, 없애지 벨트 | ✅ 조우했다, Obsidian | SuperWhisper |

**결론: SuperWhisper Standard 채택 (5:2 우세, 무료/무제한)**

## 4. 아키텍처 개요

```
┌──────────────────────────────────────────────────┐
│           QuickNoteObsidian (Swift 메뉴바 앱)     │
│                                                    │
│  ┌──────────────┐    ┌───────────────────────────┐│
│  │ 단축키 매니저  │    │ 파일 감시 (FSEvents)       ││
│  │ Ctrl+Opt+    │    │ ~/Documents/superwhisper/  ││
│  │ Cmd+I        │    │ recordings/                ││
│  └──────┬───────┘    └────────────┬──────────────┘│
│         │                         │                │
│         ▼                         ▼                │
│  SuperWhisper              meta.json 감지          │
│  녹음 트리거               & result 파싱           │
│  (딥링크 호출)                    │                │
│                                   ▼                │
│                        ┌──────────────────┐       │
│                        │ Obsidian 노트     │       │
│                        │ 생성기            │       │
│                        │ - 첫줄 → 제목     │       │
│                        │ - 나머지 → 본문   │       │
│                        └──────────────────┘       │
│                                   │                │
│                                   ▼                │
│                  ~/Documents/Obsidian Vault/       │
│                  Project/아이디어/{제목}.md         │
└──────────────────────────────────────────────────┘
```

## 5. 동작 흐름

### 5-1. 녹음 트리거
1. 사용자가 `Ctrl+Option+Cmd+I` 누름
2. 앱이 `superwhisper://record` 딥링크 호출 → SuperWhisper 녹음 시작
3. 사용자가 말하고 녹음 종료

### 5-2. 텍스트 수집
4. SuperWhisper가 `~/Documents/superwhisper/recordings/{timestamp}/meta.json` 저장
5. FSEvents가 새 meta.json 감지
6. JSON 파싱 → `result` 필드에서 텍스트 추출

### 5-3. 노트 생성
7. 텍스트 첫 줄 → 노트 제목 (파일명 불가 문자 제거)
8. 전체 텍스트 → 노트 본문
9. 프론트매터 추가 (날짜, 태그 등)
10. `~/Documents/Obsidian Vault/Project/아이디어/{제목}.md` 생성

### 5-4. 결과 표시
11. 플로팅 HUD 패널로 전체 텍스트 자동 표시 (3~5초, 클릭 시 닫힘)
12. 메뉴바 드롭다운에 최근 기록 누적 (나중에 다시 확인 가능)
13. 메뉴바 아이콘 상태 변경 (정상/이상)

## 6. 외부 의존성 리스크

SuperWhisper는 서드파티 앱이므로 업데이트 시 파일 구조가 예고 없이 변경될 수 있음.

### 방어 설계
| 상황 | 감지 방법 | 사용자 알림 |
|---|---|---|
| 감시 폴더 사라짐 | FSEvents 감시 실패 | "녹음 폴더를 찾을 수 없습니다" |
| meta.json 구조 변경 | JSON 파싱 시 키 누락 | "텍스트를 읽을 수 없습니다. SW 업데이트 확인" |
| result 빈 문자열 | 빈 문자열 체크 | "녹음 결과가 비어있습니다" |
| 폴더 경로 변경됨 | 앱 시작 시 경로 확인 | 메뉴바 경고 아이콘 |

### 설계 원칙
- 감시 폴더 경로: 하드코딩 금지 → 설정값으로 관리
- 파싱 실패 시 crash 금지 → 에러 메시지 + 메뉴바 상태 표시
- SuperWhisper 히스토리 경로는 Settings > Advanced에서 변경 가능하므로 사용자가 앱에서도 재설정할 수 있어야 함

## 7. rumps 레시피에서 가져올 교훈 (Swift 적용)

macOS_rumps_tray_app_recipe.md에서 Python으로 겪은 문제들 중 Swift에서도 주의할 것:

| rumps에서 겪은 문제 | Swift 적용 |
|---|---|
| 한글 입력기에서 `charactersIgnoringModifiers` 실패 | `event.keyCode` 사용 필수 (하드웨어 키 코드) |
| config.json 쓰기 중 crash → 파일 손상 | 원자적 저장: tmp 파일 쓰기 → rename |
| HUD 알림 패턴 (NSWindow + performSelector 지연) | 유사 패턴으로 HUD 구현 |
| GC로 이벤트 모니터 소멸 | Swift에서도 strong reference 유지 필수 |

## 8. MVP 범위

### 포함
- 메뉴바 상주
- 글로벌 단축키 → SuperWhisper 녹음 트리거 (딥링크)
- meta.json 파일 감시 → 텍스트 추출
- Obsidian 노트 자동 생성 (첫줄=제목, 전체=본문)
- 플로팅 HUD로 결과 텍스트 자동 표시 (3~5초, 클릭 시 닫힘)
- 메뉴바 드롭다운에 최근 기록 누적
- 에러 상태 메뉴바 아이콘 표시
- 감시 폴더 경로 설정 가능

### 제외 (MVP 이후, 우선순위 낮음)
- 단축키/폴더경로 세트 추가 기능
- 노트 템플릿 커스터마이징
